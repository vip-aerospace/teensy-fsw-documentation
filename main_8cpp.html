void loop ( )

Operating code. Periodically checks for incoming packets and polls the attached sensors. This function runs in a loop forever, after setup() completes.

In one iteration of the loop() function, the following main actions occur:

The main_queue is checked for a packet. If there is a packet, check its header for the destination node.

  • If the destination node is the Ground Station, check the packet's destination channel.
    • If the destination channel is the RFM23, add the packet to the RFM23 queue.
    • If the destination channel is Astrodev, add the packet to the Astrodev queue.
  • Otherwise, if the destination node is the Raspberry Pi, add the packet to the Raspberry Pi queue.
  • Otherwise, if the destination node is the Teensy, check the type of packet by inspecting its header.
    • If the packet is a Ping packet, change the packet to become a Pong reply packet, and check the destination channel of the Ping packet.
      • If the destination channel is the RFM23, add the Pong reply packet to the RFM23 queue.
      • If the destination channel is Astrodev, add the Pong reply packet to the Astrodev queue.
    • If the packet is a PDU command packet, add the packet to the PDU queue.
    • If the packet is a switch command, check the first 8 bits of the packet's data, which represents the switch being commanded.
      • If the switch to be commanded is the Raspberry Pi switch, check the current (or voltage?) of the battery board.
        • If the first two values of data in the packet's payload are '11' and the current (or voltage?) is above a certain threshold, enable the Raspberry Pi and add the Raspberry Pi thread to the thread list.
        • Otherwise if the first value of data in the packet's payload is a '0', add the packet to the Raspberry Pi queue.
      • Otherwise, add the packet to the PDU queue.
    • If the packet is a switch status command, check the first 8 bits of the packet's data, which represents the switch being polled.
      • If the switch to be polled is the Raspberry Pi switch, change the packet's payload to contain the status of the Raspberry Pi enable pin, the packet's header to be a reply, and add the reply packet to the RFM23 queue.
      • Otherwise, add the packet to the PDU queue.
    • If the packet is a beacon command, poll all the sensors.

The sensors are periodically polled and the corresponding readings updated.

The GPS is updated on every iteration.

void setup ( )

Initialization code.

Initializes the various sensors attached to the Teensy. This function is called once, upon powerup.

First, a serial connection is established. This allows the Teensy to communicate to the attached host computer over USB.

Next, the frequency of the Teensy's processor is set. Allowed frequencies in MHz are: 24, 150, 396, 450, 528, 600.

Then, the USB connection is started.

The pin that enables the Raspberry Pi is set up as an output pin.

The various sensors in the satellite are initialized. Specifically, the magnetometer, Inertial Measurement Unit (IMU), current, and GPS sensors are initialized.

Each thread that the Teensy processes is assigned a slice of computing time.

The threads are initialized. The threads are stored in a linked list called thread_list. Each thread is added to the end of this list.
