void Artemis::Teensy::Channels::pdu_channel ( )

The PDU Channel.

The channel blocks until a Serial connection can be established. After a connection is established, the channel is delayed by 5 seconds to give the PDU time to warm up.

The channel then verifies the established serial connection. It sends a Ping packet to the PDU, and if it receives a Pong reply, the connection is considered sucessfully established. It will do this test in an infinite loop until the connection is verified.

Next, the burn wire is triggered. The Teensy commands the burn wire switch on, waits 10 seconds, then commands the switch off. TODO: ASYNC DELAY TO ALLOW WDT RESET

Finally, the main operating loop for the PDU is entered. In one iteration of this loop, the following occurs:

The PDU Queue is checked for a packet. If there is a packet, check its header for its type.

  • If the packet is a CommandEpsCommunicate packet:
    • Send a Ping packet to the PDU. If a Pong packet is received, push that packet to the main queue . If a response is not received within 5 seconds, discard the ping packet.
  • If the packet is a CommandEpsSwitchName packet:
    • Set the switch in accordance with the payload of the packet.
  • If the packet is a CommandEpsSwitchStatus packet:
    • Poll the switch as defined in the packet payload.
  • If the packet is any other type, do nothing.
