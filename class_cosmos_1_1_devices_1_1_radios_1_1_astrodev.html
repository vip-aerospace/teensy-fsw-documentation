uint16_t Cosmos::Devices::Radios::Astrodev::CalcCS ( uint8_t *  data,
uint16_t  size 
)

Calculate a checksum.

Parameters
dataA pointer to the data to be calculated over.
sizeThe size of the data to be calculated over.
Returns
uint16_t A two-byte checksum of the data.

The checksum consists of two bytes, ck_a and ck_b. Combined, they form the two-byte checksum cs.

The checksum algorithm is relatively simple. For each byte in the data, it is added to ck_a. ck_a is then added to ck_b. This repeats for every byte in the data to be summed over.

The resulting two-byte checksum consists of ck_a in the low byte position and ck_b in the high byte position.

Connect to the Astrodev radio.

Returns
int32_t Returns 0 if successful, negative value if unsuccessful.

The radio is reset. The reset will be tried 5 times if unsuccessful. If the reset still fails, return with an error code. On successful reset, wait 2 seconds.

int32_t Cosmos::Devices::Radios::Astrodev::Receive ( frame &  message)

Receive an Astrodev packet from the radio.

Parameters
messageA pointer to an Astrodev packet struct to hold the recieved data from the radio.
Returns
int32_t Returns the Astrodev command type from the packet header if successful, 0 for header-only responses, negative value if unsuccessful.

The internal incoming queue mutex is locked.

The synchronization characters are awaited. The serial interface is read from, one character at a time, twice. The first syncronization character is optional, but if both synchronization characters are not read, return with an error code. This must occur within a one-second timeout period. The synchronization characters are then placed in the packet's header.

The remaining six bytes of the header are read in from the serial interface and stored in the packet's header. In the event that the serial interface does not yet have six bytes available, read all the bytes available then read the remaining bytes to form a full header. If no more bytes are available, return with an error code.

Calculate the header checksum and compare it to the received header's checksum. If they do not match, return with an error code.

The ack and sizelo bytes in the packet header are checked to determine what type of packet is incoming from the radio.

If ack is 0x0a and sizelo is 0x0a, then the incoming packet is an acknowledge response from the radio. The internal buffer full status flag is set to the value received in the reply from the radio. The command type is checked in the received packet's header. If it is a TRANSMIT packet, then the internal ack_transmit variable is set to true, indicating that the transmisssion command has been acknowldged by the radio.

◆ Transmit() [2/2]

int32_t Cosmos::Devices::Radios::Astrodev::Transmit ( frame &  message)

Transmit an Astrodev packet to the radio.

Parameters
messageThe Astrodev packet struct to be sent to the radio.
Returns
int32_t Returns size of transmitted packet if successful, negative value if unsuccessful.

The internal outgoing queue mutex is locked.

The size of the payload is checked. If it is too large to be sent, return an error value.

Check the type of packet from its header data. If the packet is a TRANSMIT packet, containing data to be sent to the ground station, an acknowledgement reply from the radio is not necessary.

Fill in the header data with the synchronization bytes. Additionally specify that is packet is being sent from the Teensy to the radio, empty the high byte of the packet size field (only the low byte is used), and calculate the checksum bytes.

Send the header data over the serial connection. If the header is the entire packet, then skip the payload processing and return the length of the packet.

Define the two-byte payload checksum using a union. The checksum is calculated over all bytes in the packet including the header with the exception of the two synchronization bytes. The checksum is then appended to the end of the payload.

Each byte of the packet's payload is sent over the serial connection.

The size of the packet is returned as the size of the payload, plus 8 bytes for the header and two bytes for the checksum.
